---

## 🎯 Purpose

**ohada-lib v0.0.1** is a **developer-first library** designed to simplify accounting software development in OHADA-compliant countries.  

The library transforms **business events** (sales, purchases, expenses) into **fully-formed OHADA-compliant journal entries** and **stock movements**, freeing developers from the complexity of SYSCOHADA rules.

---

## 🔹 Key Features — v0.0.1

### 1️⃣ Sale Operation
- Records the sale of goods or services
- Supports **cash, bank, mixed, or partial payments**
- Automatically generates:
  - OHADA-compliant **journal entries**
  - **Stock decrease** for sold goods
- Example use cases:
  - Point-of-sale transactions
  - Customer invoicing
  - Partial payment handling

### 2️⃣ Purchase Operation
- Records the acquisition of goods or services
- Supports **cash, bank, mixed, or partial payments**
- Automatically generates:
  - OHADA-compliant **journal entries**
  - **Stock increase** for purchased goods
- Example use cases:
  - Supplier invoice recording
  - Stock intake

### 3️⃣ Expense Operation
- Records non-stock expenses (rent, transport, electricity)
- Generates OHADA-compliant **journal entries**
- No stock movement
- Can be linked to sales/purchases or standalone

---

## 🔹 Core Principles

1. **Dev-first DX**
   - Developers never create journal entries manually
   - Only emit **business events** (`SaleRequest`, `PurchaseRequest`, `ExpenseRequest`)
   - Engine decides accounts and amounts

2. **OHADA Defaults**
   - Default account mapping matches **SYSCOHADA 2017**
   - Developers can override the account selection with **autocomplete validation**

3. **Stock Movement**
   - Automatically tracked for Sale and Purchase operations
   - Represents **IN / OUT movements** in stock accounts (Class 3)

4. **Immutable Journal**
   - Journal entries are **immutable**
   - Any corrections create **reversing entries** (full support in future versions)

5. **Extensible**
   - JSON-based **chart of accounts** included
   - Easy to update or extend per country or internal needs

## 🔹 Example Usage (TypeScript)

```ts
import { Ohada } from "ohada-lib";

const ohada = new Ohada();

// Record a sale
const sale = ohada.sale({
  customerId: "C001",
  items: [
    { sku: "P001", quantity: 5, unitPrice: 100 }
  ],
  payment: { cash: 50, bank: 250 }
});

// Record a purchase
const purchase = ohada.purchase({
  supplierId: "S001",
  items: [
    { sku: "P002", quantity: 10, unitPrice: 50 }
  ],
  payment: { bank: 400 }
});

// Record an expense
const expense = ohada.expense({
  label: "Transport fees",
  amount: 60,
  paymentMethod: "cash"
});

````

**What the engine returns:**

* `journalEntries`: fully balanced OHADA entries
* `stockMovements`: IN/OUT movements
* `totals`: aggregated debit/credit amounts

## 🔹 v0.0.1 Limitations

* No payroll support
* No VAT automation
* No depreciation/asset module
* No multi-currency handling
* Simplified stock: real-time quantity only, no FIFO/LIFO
* No reporting (Balance Sheet, General Ledger, etc.) — to come in v0.0.2
